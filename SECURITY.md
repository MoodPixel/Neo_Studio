# Security Policy

## 🛡️ Supported Versions

Neo Studio is currently in **active development (V1)**.

At this stage:

* Only the **latest version (main branch)** is actively maintained
* Older versions are **not supported** for security updates

---

## 🚨 Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

### 📩 How to report

* Open a **private security advisory** on GitHub (recommended), or
* Contact the maintainer directly:

```text
lakmalsirimanna95@gmail.com
```

---

## ⚠️ What to include

Please provide:

* A clear description of the issue
* Steps to reproduce
* Potential impact
* Screenshots or logs (if applicable)

---

## ⏱️ Response Time

* Initial response: **within a few days**
* Fix timeline: depends on severity and complexity

---

## 🔒 Responsible Disclosure

Please:

* Do **not publicly disclose** the vulnerability before it is fixed
* Allow time for the issue to be reviewed and resolved

---

## ❌ What is NOT considered a vulnerability

* Issues caused by external backends (ComfyUI, KoboldCPP, etc.)
* Local environment misconfigurations
* Missing third-party dependencies

Neo Studio acts as a **frontend control system**, and security of external tools is handled by their respective projects.

---

## 🧠 Scope

Security reports should focus on:

* Neo Studio application behavior
* Backend launcher handling
* File handling or data exposure
* Local execution risks

---

## 🙌 Final Note

Security is important, even in local-first tools.

Thank you for helping keep Neo Studio safe.
